Machine Learning through Scikit-learn with the K-nearest neighbors


K-nearest neighbors, is an example of instance-based learning where we instead simply store the training data and use it to make new predictions. In general, instance-based techniques such as k-nearest neighbors are lazy learners, as compared to model-based techniques which are eager learners. A lazy approach will only "learn" from the data (to make a prediction) when a new query is made while an eager learner will learn from the data right away and build a generalized model capable of predicting any value. Thus, lazy learners are fast to train and slower to query, while eager learners are slower to train but can make new predictions very quickly.

Machine learning was implemented by the K-nearest neighbors model through Scikit-learn. K-nearest neighbors is a simple model that creates labels with training data and then predicts a label. It finds a predefined number of training samples closest in distance to the new point and uses that to predict the label. The predefined number is also known as K and needs to be odd in order to prevent ties and create an unambiguous result. K-nearest neighbors is especially useful and accurate for smaller sets of data, which fits our data set particularly well.

Implementation of K-nearest neighbors model

Get the orignal data and encode it so can be feed into ML

The original data was divided into two groups: training and testing.

Training was completed with the training data and was stored in the Models folder using pickle. The model was then tested with the test data and resulted in an accuracy of 0.969.
Once the model was tested and proven to be accurate, the customer data, also known as validation data, can be fed into the model, which was not seen before and a prediction can be obtained. At the moment, the customer data is not defined by user input, but in the future, it will be.
