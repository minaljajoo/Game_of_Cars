GAME OF CARS: BEHIND THE SCENES

How it Works...

Hand-Picked Categorical Information

After a long exploration period, we agreed on a strategy that would enable us to harness the features of cars, while also injecting unique characteristics of different cars within those styles. As a group, we researched and rated car styles on a handful of characteristics for each car style, namely: Resale Cat,fuel type,vehicle class,trans, drive,smog rating,city_mpg,hwy_mpg,cmb_mpg,Greenhouse gas score, smartway, comb CO2, Sales 2016, Sales 2017,Sales 2018 and Loan.

Scrape, Scrape, and Scrape Some More



We scrape the data form different source and make our own data base.

App.py: Restful API



Calls restful apis to connect all HTML and JavaScript files to run the webpage by providing all the routes in the webpage.

HTML and JavaScript


A frontend for Game of Cars, which is user friendly through the use of JavaScript and CSS styling. The JavaScript allows for animations, while the CSS allows for styling to make the webpages look better for the user. Bootstrap was the particular style used.

Jupyter Notebook


Jupyter Notebook was used to clean the raw data and store it into csv files. It was also used to test the models.

SQL/SQLite:


The cleaned data was put into four tables and then stored into a database through SQLite. One table was the original data and another table was the validation data. The other two tables were those two tables converted into integers.

Drumroll, Please

Put everything together, and we have an interactive site which allows the user to select one of the name, and then runs a machine learning algorithm to determine which type loan the user can get and then allow it to pick one car from the lsit and call other machine learning algorithms and show the resale value of car, combine milage as low, high or medium and also show if the vehicle is smartway or not.
